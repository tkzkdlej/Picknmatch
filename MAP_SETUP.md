# 지도 API 설정 가이드

오시는길 페이지에 카카오맵이 표시되려면 **JavaScript 키**와 **도메인 등록**이 필요합니다.

---

## 카카오맵 키 발급 (권장, 무료)

1. **https://developers.kakao.com/** 접속 → 카카오 계정 로그인
2. **[내 애플리케이션]** → **[애플리케이션 추가하기]**
3. 앱 이름 입력 (예: 픽앤매치) 후 저장
4. 생성한 앱 클릭 → **[플랫폼]** 탭
5. **[Web 플랫폼 등록]** 클릭
   - **브라우저 주소창에 보이는 URL을 그대로** 등록하세요 (포트 번호까지 정확히!)
   - 예: `http://127.0.0.1:5500` (Live Server), `http://localhost:3000`, `file://` 불가
   - **Vercel 미리보기/배포:** 반드시 아래를 **추가** 등록하세요 (HTTPS, 슬래시 없이).
     - `https://picknmatch.vercel.app`
     - (나중에 연결하는) 구매한 도메인 예: `https://www.example.com` → 콘솔에 `https://www.example.com` 형태로 등록
   - 도메인을 바꾸거나 새로 쓰면 카카오 콘솔에 등록하지 않으면 지도가 **“로딩 중”에 멈추거나** 타일이 안 뜰 수 있습니다.
6. **[앱 키]** 탭에서 **JavaScript 키** 복사
7. **[제품 설정]** → **[카카오맵]** → **활성화 상태 ON** 확인
8. `map-config.js` 파일 열어서 `YOUR_KAKAO_MAP_APP_KEY`를 복사한 키로 교체

### 운영 배포 전 보안 체크(중요)
- **JavaScript 키는 브라우저로 내려가므로 “노출되지 않게” 할 수 없습니다.** 대신 반드시 아래처럼 제한해서 악용을 막습니다.
- **도메인 제한**: 카카오 개발자 콘솔 **[플랫폼] → [Web]**에 운영 도메인(예: `https://www.example.com`)을 정확히 등록
- **키 분리 권장**: 개발용(로컬/프리뷰)과 운영용을 **서로 다른 앱/키로 분리**해서 관리(운영 키를 로컬에 넣지 않기)
- **키 로테이션**: 이미 공유/노출이 의심되면 새 키로 교체하고 기존 키는 폐기
- **쿼터/모니터링**: 콘솔에서 호출량(쿼터)과 오류를 주기적으로 확인

### 지도가 하얗게만 나올 때
- **도메인 미등록**이 가장 흔한 원인입니다.
- 브라우저 주소창 URL(예: `http://127.0.0.1:5500/...`)에서 **포트까지 포함한 도메인**을 플랫폼에 등록하세요.
  - 예: `http://127.0.0.1:5500` 또는 `http://localhost:5500`
- `file://` 로 열면 동작하지 않습니다. 반드시 **로컬 서버**로 실행하세요.

**로컬 서버 실행 방법**
- VS Code/Cursor: Live Server 확장 설치 후 우클릭 → Open with Live Server (보통 `http://127.0.0.1:5500`)
- 터미널: `talent-search` 폴더에서 `npx serve -l 3000` 실행 후 `http://localhost:3000` 접속

```js
window.KAKAO_MAP_APP_KEY = "발급받은_JavaScript_키";
```

---

## API 키 없이 사용하기

키를 설정하지 않아도 **지도 영역 클릭 시 카카오맵이 새 탭에서 열립니다.**  
다만 페이지 안에 바로 지도를 띄우려면 위 과정으로 키 발급이 필요합니다.

## (참고) 네이버 지도 API 인증 실패 시 확인사항

이전에 네이버 지도를 사용하다 인증 실패가 났다면, 아래를 확인하세요.

1. **네이버 클라우드 플랫폼** https://console.ncloud.com/maps/application
2. 애플리케이션에서 **Web 서비스 URL** 등록
   - 포트·경로 제외: `http://127.0.0.1/main` → `http://127.0.0.1`
   - `file://` 로 열면 동작하지 않음 → 로컬 서버 사용 필요
3. **Dynamic Map** 서비스가 활성화되어 있는지 확인
4. 스크립트 파라미터는 `ncpKeyId` 사용 (구버전 `ncpClientId` 사용 금지)
